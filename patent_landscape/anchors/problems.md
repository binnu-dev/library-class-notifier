# problems.md (v1, P1~P8 / 각 P = 문제 + 이웃 산업)

> 렌즈 B — 이종산업 차용의 앵커. 사람이 손으로 편집하는 단일 출처(source of truth).
> 각 P에 박힌 이웃 산업이 곧 검색 사전필터(CPC)다.

```
P1  고밀도 셀 스택 방열          ← 데이터센터 냉각 / LED / 전력반도체
P2  NVH 저감                    ← 항공 캐빈 / HVAC / 가전
P3  경량화 (강성 손실 없이)      ← 항공우주 / 스포츠장비 / 건축
P4  악조건 퍼셉션 (비·안개·역광)  ← 의료영상 / 국방 / 농업드론
P5  체결부 내구·피로            ← 풍력터빈 / 철도 / 교량
P6  커넥터·하우징 밀폐/방수       ← 해양장비 / 의료기기 / 가전
P7  협소공간 하네스·부품 라우팅    ← 로보틱스 / 웨어러블 / 내시경
P8  실내공간 활용·모듈러 패키징    ← 항공 갤리·시트 / RV·캠핑카 / 요트 / 건축 모듈러
백로그:  인라인 결함 검출
```

## 구현 메모

- 위 목록은 **사람이 읽는 앵커**다. 각 P의 (a) 임베딩 매칭용 문장과 (b) 이웃 산업을
  좁히는 CPC 사전필터 코드는 `patent_landscape/registry.py`의 `PROBLEMS`에 정의한다.
- 매칭 파이프라인: 이웃 산업 → CPC 사전필터로 우주 좁힘 → 그 안에서 Pn 문장 임베딩 ↔
  특허 abstract 임베딩 유사도 top-k. 단어가 달라도 구조적으로 닮은 것을 잡기 위함.
- 백로그(인라인 결함 검출)는 v1 범위 밖. `registry.py`에 `backlog=True`로 표시만.
